# AIと一緒にウェブサイトを作ってみた話

今日、Claude Code というツールを使って、フランス旅行の準備に役立つウェブサイトを一から作りました。Claude Code は Anthropic という会社が作った AI アシスタントで、ターミナル（パソコンの黒い画面）から直接対話しながらプログラミングができるものです。この記事では、実際にどんなふうに作業が進んだのかを振り返ってみます。

## きっかけ

フランス旅行を控えて、必要な情報をひとつの場所にまとめたいと思いました。パスポートのこと、飛行機のこと、ホテルの選び方、持っていくもの、現地での通信手段、保険、そして簡単なフランス語。こういった情報はネット上にたくさんありますが、自分用に整理されたページがあると便利です。

## 最初の一歩：トップページ

Claude に「フランス旅行準備のガイドサイトを作りたい」と伝えたところ、まず HTML と CSS のファイルが提案されました。HTML というのはウェブページの骨組みを作る言語で、CSS はその見た目を整える言語です。

できあがったトップページには、パスポートやビザの注意点、航空券の選び方、パリの宿泊エリアの特徴、持ち物チェックリスト、Wi-Fi の選択肢、旅行保険の選び方、基本的なフランス語フレーズがカード形式で並びました。フランスの国旗をイメージした青を基調にしたデザインで、すっきり読みやすい仕上がりでした。

## 会話集ページ：実践的なフレンチ

次に作ったのが、シチュエーション別の会話集です。カフェで注文するとき、ホテルにチェックインするとき、お店で買い物するとき、道に迷ったとき、誰かと知り合ったとき、そして困ったとき。6つの場面を用意しました。

このページの面白いところは「派生パターン」という仕組みです。たとえばカフェの基本の注文会話から、「おすすめを聞く」「お会計を頼む」「料理の感想を伝える」といった分岐が用意されています。実際の旅行では会話が一本道で終わることはないので、こうした枝分かれがあると実践的に練習できます。

会話はチャットの吹き出しのような見た目で表示されます。左側がフランス人の相手、右側が自分。フランス語、英語、日本語の3つが並んで書かれているので、意味を確認しながら読み進められます。

## 旅行プランナー：自動でプランを組む

3つ目に作ったのが、旅行プランを自動生成するページです。行き先（パリ、リヨン、マルセイユなど）と日数と興味のあるジャンル（美術、グルメ、歴史など）を選んでボタンを押すと、一日ごとに午前・午後・夜の予定が組み上がります。

この機能は JavaScript というプログラミング言語で動いています。あらかじめ用意された観光スポットのデータの中から、選んだ条件に合うものをスコアリング（点数づけ）して、高い順に割り振る仕組みです。「別のプランを見る」ボタンを押せば、ランダム要素が入っているので違うプランが出てきます。

ここまでの作業を Claude は3回のやりとりでこなしました。こちらが「次はこういうページがほしい」と伝えると、ファイルを作り、既存のページにリンクを追加し、スタイルを統一する、という一連の作業をまとめてやってくれます。

## デザイン改善：途中まで

3ページができた時点で、いくつか気になるところが出てきました。ページごとにナビゲーション（ページ上部のリンク一覧）の構成がバラバラだったこと、一部の装飾が HTML ファイルに直接書き込まれていたこと、スマートフォンでの表示がもう少し改善できそうだったことです。

そこで Claude に改善を依頼しました。Claude はまず現状を分析して、改善計画を立てました。ナビゲーションを「ページ間の移動」と「ページ内の移動」の二段構えにすること、HTML 内の装飾指定を CSS ファイルにまとめること、画面サイズごとの表示を3段階（デスクトップ、タブレット、スマートフォン）で調整すること、カードにマウスを乗せたときのアニメーションを追加すること、視覚効果を減らしたい人への配慮を入れること。

この改善作業は CSS ファイルの書き換えから始まりましたが、途中で止めて今この記事を書いています。

## Claude Code を使ってみて感じたこと

一番印象的だったのは、対話のテンポのよさです。「こういうものが作りたい」と伝えると、ファイル構成から中身のテキストまで一気に作ってくれます。プログラミングの知識がある人なら、出来上がったコードを見て「ここはこう変えたい」と細かく指示できますし、知識がなくても「もう少し見やすくして」「スマホで見たときに崩れないようにして」といった伝え方で十分通じます。

もうひとつ感じたのは、既存のファイルを読んでから提案してくれるという点です。2ページ目や3ページ目を作るとき、Claude は最初に作ったファイルを読み直して、同じデザインの流れを引き継いでくれました。色の使い方やカードの角丸の大きさ、文字の大きさなど、細かいところが自然に揃います。

ただし万能ではありません。たとえば「デザインを改善して」という大きなお題に対しては、こちらが計画を確認して「これでいこう」と承認するステップが入ります。勝手にどんどん変えていくのではなく、一度立ち止まって方針を見せてくれるのは安心感があります。

## まとめ

半日ほどの作業で、3ページ構成のウェブサイトができました。パスポート情報から実践的なフランス語会話、旅行プラン自動生成まで、旅行準備に必要なものがひとまとまりになっています。デザインの改善はまだ途中ですが、骨格はしっかりできているので、ここから先は少しずつ仕上げていけそうです。

AI と一緒にものを作るというのは、優秀なアシスタントに隣にいてもらいながら作業するような感覚でした。道具の使い方を知っていれば早く進みますが、知らなくても「こんな感じにしたい」で伝わる。そこが Claude Code の良いところだと思います。
